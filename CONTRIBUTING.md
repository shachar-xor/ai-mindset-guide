# Contributing to AI Usage Rules & Tips

Thank you for your interest in contributing! This guide is meant to be a living document that evolves based on real-world experience.

## How You Can Contribute

### 1. Share Your Examples

Have a real-world scenario that illustrates these principles? We'd love to add it!

**What makes a good example:**
- Real situation you encountered
- Shows how a specific rule or tip helped
- Includes before/after or the thought process
- Relatable to other engineers

**How to contribute:**
- Open an issue with the "example" label
- Or submit a PR adding to `EXAMPLES.md`

### 2. Improve the Framework

Found a way to explain something better? Have a suggestion for improvement?

**What we're looking for:**
- Clearer explanations
- Better examples
- Additional insights
- Corrections or clarifications

**How to contribute:**
- Open an issue with your suggestion
- Or submit a PR with the changes

### 3. Add to the FAQ

Have a question that should be in the FAQ?

**How to contribute:**
- Open an issue with the "question" label
- We'll add it to the FAQ with an answer

### 4. Translate

Want to make this guide available in other languages?

**How to contribute:**
- Open an issue to coordinate (avoid duplicate work)
- Create a new directory: `translations/[language-code]/`
- Translate all markdown files
- Submit a PR

**Translation guidelines:**
- Keep the structure and formatting
- Adapt examples to be culturally relevant if needed
- Maintain the tone and philosophy

### 5. Report Issues

Found a typo, broken link, or error?

**How to contribute:**
- Open an issue with details
- Or submit a PR with the fix

## Contribution Guidelines

### Style Guide

**Tone:**
- Conversational and practical
- Honest and direct
- Avoid jargon when possible
- Use examples liberally

**Structure:**
- Start with TL;DR or key insight
- Provide context and explanation
- Include concrete examples
- End with actionable takeaways

**Formatting:**
- Use markdown formatting consistently
- Include code blocks for prompts/examples

- Keep lines under 120 characters when possible

### What We're NOT Looking For

**Please avoid:**
- Tool-specific tactics (keep it universal)
- Overly technical prompt engineering techniques
- Marketing or promotional content
- Dramatic changes to the core philosophy

This guide is about **mindset and principles**, not techniques and tactics.

## Pull Request Process

1. **Fork the repository**
2. **Create a branch** for your changes
3. **Make your changes** following the style guide
4. **Test your changes** (check markdown rendering)
5. **Submit a PR** with a clear description

**PR Description should include:**
- What you changed and why
- How it improves the guide
- Any relevant context

## Code of Conduct

### Our Standards

- Be respectful and constructive
- Focus on what's best for the community
- Show empathy toward others
- Accept constructive criticism gracefully

### Unacceptable Behavior

- Harassment or discriminatory language
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information

## Questions?

Not sure if your contribution fits? **Just ask!**

- Open an issue with your question
- Tag it with "question"
- We'll help you figure it out

## Recognition

Contributors will be recognized in the README. Thank you for helping make this guide better!
